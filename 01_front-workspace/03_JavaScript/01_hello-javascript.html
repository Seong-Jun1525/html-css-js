자바스크립트 개요

스크립트 언어란?

프로그램의 동작을 사용자 요구에 맞게 수행하도록 해주는 용도
빠르게 배울 수 있고 짧은 소스 코드로 상호작용 되도록 고안됨

자바스크립트

웹 브라우저에서 많이 사용되는 인터프리터 방식의 스크립트 언어
→ 인터프리터 방식 : 코드를 한 줄씩 읽어가면서 바로 실행하는 방식
→ 코드에 문법 오류가 있으면 실행 시점에 알 수 있음

장단점

장점

  • 컴파일 과정 없이 인터프리터로 코드를 한 줄씩 읽어가면서 바로 실행되기 때문에 수행 속도가 빠름
  • 코드 작성이 간단해서 처음 접하기 수월하다(단순한 구조/원칙/규칙)
    ex) 접근제한자 개념 없음, 변수 선언 시 자료형을 정해줄 필요 없음

단점

  • 웹에 특화된 기술 → 내부에서 제공하는 기능이 제한적
  • HTML 문서 내에 작성하면 외부에 공개 됨 → 보안 취약점 발생

자바스크립트 사용법

  1. 인라인 방식 : 태그 내에 간단한 소스코드 작성
  2. 내부 스크립트 방식 : HTML 문서 내에 소스코드를 작성 script 태그
  3. 외부 스크립트 방식 : 별도의 자바스크립트 파일로 소스코드를 작성해서 현재 문서에 연결해서 사용하는 방법

인라인 방식

[표현법] <태그명 on이벤트명="자바스크립트 코드(이벤트가 발생되었을 때 실행할 내용)">

내부 방식

script 태그 영역 안에 소스코드를 작성
→ head 태그, body 태그 내에 모두 작성 가능

외부 방식

외부 파일 작성 후 현재 문서에 연결
→ script 태그를 사용하여 src 속성에 파일 경로를 작성
